# RAG 平台侧已完成工作与后续扩展说明

本文档用于说明当前 RAG 权限控制平台的完成情况，以及后续可继续扩展的 AI 生成层工作。

---

## 一、当前结论

截至当前版本，平台侧权限控制与 RAG 检索治理闭环已经基本完成。

当前已完成的核心链路为：

    基础信息管理
    → 权限策略管理
    → 策略绑定
    → 文档权限标签
    → RAG 文件入库
    → 访问请求处理
    → 向量检索联调
    → 权限二次过滤
    → 审计日志
    → 日志导出
    → 行为分析告警

## 三、日志导出完成情况

RAG 审计日志已支持 Excel 导出。

审计日志记录内容包括：

- 用户信息；
- 检索内容；
- 用户组；
- 权限标签；
- metadata filter；
- 原始候选结果；
- 通过结果；
- 拦截结果；
- 最终响应；
- 访问决策；
- 检索耗时。

管理员可以在 RAG 审计日志页面查看日志详情，并导出 Excel 文件。

---

## 四、行为分析完成情况

系统已新增 RAG 行为告警模块。

行为分析模块基于 sys_rag_audit_log 审计日志生成告警记录，并写入 sys_rag_behavior_alert 表。

当前支持：

- 普通访问记录；
- 权限拒绝访问；
- 二次过滤拦截；
- 异常耗时访问。

管理员可以在 RAG 行为告警页面查看、筛选、删除和导出告警记录。

---

## 五、当前边界

当前项目已经完成平台侧主要功能，但完整的大模型生成回答层仍属于后续扩展内容。

当前系统已经具备：

- RAG 文件入库；
- RAG 检索联调；
- 权限过滤；
- 审计留痕；
- 日志导出；
- 行为分析。

后续如果要形成完整的大模型问答系统，还需要继续接入：

- LLM API 或本地大模型；
- Prompt 拼接模块；
- 基于检索结果的自然语言答案生成；
- 回答引用追踪；
- 大模型输出侧权限审计。
