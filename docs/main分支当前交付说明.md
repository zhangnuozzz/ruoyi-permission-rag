# main 分支当前交付说明

本文档说明当前 main 分支的主要交付内容。

---

## 一、当前分支状态

当前 main 分支已经完成权限控制 RAG 平台侧主要功能。

平台侧已经形成以下闭环：

    权限治理
    → 文档权限标签
    → RAG 文件入库
    → RAG 检索
    → 权限二次过滤
    → 审计日志
    → 日志导出
    → 行为分析告警

---

## 二、主要交付功能

### 1. 权限治理模块

已完成：

- 用户组管理；
- 用户与用户组关系管理；
- 权限策略管理；
- 策略绑定管理；
- 文档权限标签管理。

### 2. RAG 文件入库模块

已完成：

- 文件上传页面；
- 平台侧转发 RAG Server；
- MariaDB 文件元数据写入；
- MinIO 原文件存储；
- Milvus 向量切块写入；
- 文档权限标签联动生成。

### 3. RAG 安全检索模块

已完成：

- 用户权限上下文生成；
- metadata filter 构造；
- 远程 RAG Server 检索调用；
- 检索结果二次权限过滤；
- 访问决策生成。

### 4. RAG 审计日志模块

已完成：

- 审计日志入库；
- JSON 级别链路留痕；
- 日志详情查看；
- 审计日志 Excel 导出。

JSON 留痕内容包括：

- 用户上下文；
- 请求 JSON；
- 原始候选结果；
- 通过结果；
- 拦截结果；
- 最终响应。

### 5. RAG 行为分析模块

已完成：

- 行为告警表；
- 行为分析服务；
- 行为告警页面；
- 行为分析按钮；
- 告警导出；
- 告警删除。

当前支持：

- 普通访问记录；
- 权限拒绝访问；
- 二次过滤拦截；
- 异常耗时访问。

### 6. 首页总览

已完成首页改版，首页展示：

- 平台定位；
- 系统架构图；
- 1–10 功能模块完成情况；
- 平台侧实现边界；
- 安全检索链路；
- 审计与行为分析闭环。

## 三、启动方式

启动：

    rag-start

关闭：

    rag-stop

访问：

    http://localhost:1024
